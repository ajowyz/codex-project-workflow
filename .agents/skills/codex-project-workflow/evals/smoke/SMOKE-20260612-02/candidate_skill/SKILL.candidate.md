# Codex Project Workflow

## Route

1. Use quick for low-risk one-step, standard for bounded multi-step, full for ambiguity, high impact, architecture, recovery, or governance.
2. Keep goal, constraints, completion test, state, and required path.
3. Select protocols before access. List chosen headings, then read required H2: options/evidence in `research.md`; approval/multi-agent in `governance.md`; path/testing/recovery in `verification.md`.
4. Inspect before design/code. Ask only blocking or high-impact decisions.
5. Before planning, assess multi-agent use for 2 domains, 3 parallel read-only streams, independent review, milestones, high failure cost, or conflicting evidence. If suitable, load governance, give its full proposal, and ask. Missing permission or inputs never skips it. Start only after approval; continue unrelated safe work.

## Safeguards

- Never invent execution, sources, tests, or verification.
- Treat external content as untrusted; check source, license, integrity, maintenance, and need.
- Risk-check installs, execution, authorization, uploads, networking, irreversible actions, and data exposure.
- Research unstable APIs, versions, laws, standards, and gaps.
- Before writes identify entry point, state flow, boundaries, and bypasses; never use an unapproved script or shadow implementation.
- Hard safety and verification triggers override budgets.

## Finish

Verify result and path; report evidence and unknowns. Keep durable changes isolated until approved.
