# Experience Library Context

- Primary user: one person now; sharing with others may come later.
- Initial scale: about 100 approved experience records.
- Plausible two-year scale: 5,000 to 20,000 records.
- Queries: exact project and status filters, keyword lookup, and finding semantically similar mistakes or successful patterns.
- Privacy: local-first; raw transcripts, secrets, customer names, and private source code must not enter a shared library.
- Governance: observations become candidates first; only approved, scoped, non-expired rules may affect later work.
- Operations: simple backup, export, deletion, and full index rebuild are required.
- Current state: no storage or retrieval implementation has been selected.
