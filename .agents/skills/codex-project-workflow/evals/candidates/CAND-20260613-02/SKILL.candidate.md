# Codex Project Workflow

## Route

1. Inspect first. Keep goal/constraints/completion test; ask only blockers.
2. Run `python .agents/skills/codex-project-workflow/scripts/read_reference.py NAME "Execution Rules" "Output Requirements"`. Load only triggers: `research` for unstable evidence/options; `governance` only for third-party/download, install/external execution, private networking, agents, rule activation, or approval-bound/high-impact actions; `verification` for writes, recovery, product/core path, or path proof.
3. Never scan skill dirs/read refs directly. Complete helper output counts despite nonzero exit; do not repeat it. Report incomplete failure; retry the missing protocol only. No gate is waived.
4. Before executing three streams or conflict/high-cost work, load governance; propose roles/count, ownership, main/parallel work, benefit/cost/risk, fallback; ask. Acceptance starts agents; no decision stays `proposed`; refusal/unavailability selects fallback.

## Safeguards

- Never invent execution, sources, tests, or verification.
- Before writes map entry, state, owners/bypasses. Change behavior in its existing owner; freeze an evidence-bound contract for routing, persistence/schema/migration, architecture/ownership, core capability, alternate output, path proof, or high impact. Never borrow neighboring/shadow implementations.
- Safety, authorization, and verification override budgets.

## Finish

Verify result/path. Keep durable changes isolated until approved.
