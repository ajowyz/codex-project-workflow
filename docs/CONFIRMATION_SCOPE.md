## 4. 生效规则

- 确认请求必须引用一个仍然有效的候选审计 ID。
- 审计清单内任一文件哈希变化后，本清单对应的 `待确认候选` 结论失效。
- 用户可以分组表达意见，但本次架构基线只有在所有范围内条目形成一致组合后才整体生效。
- 修改 FR、ADR 或文档治理状态时，记录确认日期和用户实际接受的编号。
- 用户确认后的状态变化必须生成确认凭证；规范化哈希与父候选审计不一致时，视为实质修改并重新审计。
