# 常用任务模板

下面模板可以直接复制到启用 `codex-project-workflow` 插件的新 Codex 线程里使用。

## 新项目启动

```text
我要启动一个新项目。请先做需求访谈和风险判断，不要直接给最终方案。

请输出：
1. 你理解的项目目标。
2. 还缺哪些关键信息。
3. 初步风险和边界。
4. 建议的下一步协作方式。
```

## 继续推进当前项目

```text
继续推进当前项目。先复核当前状态、计划文档和 git 状态，再判断下一步。

要求：
- 如果可以直接做，就直接推进。
- 如果存在高影响决定，先说明选项、影响和你的建议。
- 不要重复问已经明确授权的低风险本地读写和验证。
```

## 需要专业方案

```text
这个任务需要专业判断。请先说明：
1. 你会查哪些资料或方法论。
2. 你会比较哪些方案。
3. 每个方案的适用条件、成本、风险。
4. 你的推荐方案和不推荐其他方案的原因。

如果需要联网，请先给出研究范围和拟查询内容。
```

## 防止实现路径跑偏

```text
完成后请证明不是只生成了正确输出，而是通过项目原有入口和 owner 路径实现。

最终报告请包含：
- 用户入口是什么。
- owner 模块或函数是什么。
- 改动是否发生在 owner 路径。
- 有没有临时脚本、测试专用逻辑或直接写输出文件的旁路。
- 结果验证和路径验证分别是什么。
```

## 少打断但不越界

```text
本阶段本地只读检查、非破坏性文件修改、已有测试、文档更新和本地提交不用再问。

遇到以下情况再停下来确认：
- 联网发送项目上下文。
- 安装依赖或执行外部脚本。
- 删除、迁移、覆盖或不可逆修改。
- 高影响产品或架构决定。
- 启动多 Agent。
- 把一次经验升级为长期规则或插件行为。
```

## 多 Agent 判断

```text
请判断这个任务是否值得启用多 Agent。

如果值得，请给出：
- 每个 Agent 的职责。
- 文件和信息边界。
- 禁止事项。
- 预期收益和成本。
- 单 Agent fallback。

在我明确同意前，不要启动多 Agent。
```

## 联网研究

```text
这个任务需要查最新资料。请先给出研究批准包：
- 研究目的。
- 拟查询内容。
- 来源类型。
- 会发送哪些项目相关信息。
- 本阶段只提取哪些字段。

等我确认后再联网。
```

## 任务结束验收

```text
请按验收清单总结本次工作：
1. 目标是否完成。
2. 范围是否保持。
3. 改了哪些文件。
4. 结果怎么验证。
5. 实现路径怎么验证。
6. 剩余风险是什么。
7. 哪些内容值得沉淀为文档、测试或候选改进。
```

## 插件复验

```text
请做 codex-project-workflow 插件安装 smoke 验收：
1. 确认当前加载的技能路径。
2. 确认是否来自插件 cache，而不是项目 .agents。
3. 使用插件 helper 读取 governance、research、verification 的 Execution Rules 和 Output Requirements。
4. 报告 metrics 和失败点。
5. 不修改文件。
```

## 模型或上下文升级

```text
当前模型或上下文能力变化了。请先按项目策略评估是否需要调整工作流。

要求：
- 不要直接放宽安全、联网、安装、实现路径和长期规则确认门。
- 先说明可能受影响的任务档位。
- 给出需要回归验证的样例。
- 通过验证后再建议调整。
```
